Exercise 1 - Implementing the Singleton Pattern

This exercise demonstrates the Singleton pattern with a simple `Logger` class.

Files:
- `Logger.java` - Singleton implementation (Bill Pugh holder pattern)
- `Main.java` - Test harness that gets the instance twice and compares them

Compile and run (from project root):

```bash
javac -d out "Exercise 1 - Implementing the Singleton Pattern"/*.java
java -cp out Main
```

Expected output:
- Two log lines showing identical instance hash codes
- A final line `Same instance? true`

Note: You requested a folder name with a colon (`Exercise 1: Implementing the Singleton Pattern`). Colons are not allowed in Windows filenames, so the folder uses ` - ` instead.
